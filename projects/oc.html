Dispatch Management System

Updated: March 2022

React & Typescript Report-Tools & Python & echart DB2 & Oracle SQL/PLSQL

Project Overview

The project centered around the development and enhancement of a comprehensive dispatch system designed to optimize data management and reporting mechanisms within a logistical framework.

By implementing a structured three-tier data warehouse architecture, the system was tailored for efficient offline data handling and report generation.

Key to the project was the integration of SQL stored procedures, which streamlined the process of data organization and report development.

Contributions

In the dispatch system, I primarily utilized a three-tier data warehouse architecture and wrote SQL stored procedures for offline data processing.

This approach enabled me to successfully organize warehouse data and develop reports.

I also accomplished real-time data processing through interface protocols and real-time components such as Kafka and Flink.

The results were displayed in real-time using ECharts or reporting tools, showcasing live data presentations of inventory levels and order contracts.

Teach Stack

SQL Stored Procedures: By writing efficient SQL stored procedures for offline data processing, we were able to organize data and develop reports effectively. This approach enhances the efficiency and accuracy of data processing.

Three-tier Data Warehouse Architecture: The adoption of a layered data warehouse design optimized data storage, management, and access. This architecture supports efficient data integration and analysis to meet diverse business requirements.

Kafka: As a high-throughput distributed messaging system, Kafka was used for processing real-time data streams. It supports large-scale data processing, ensuring reliable data transmission and storage..

ECharts: An open-source visualization library based on JavaScript, ECharts was used for real-time data charting. It supports various chart types and offers rich configuration options, making data visualization more intuitive and interactive.

Interface Protocols: The project utilized standardized interface protocols to facilitate data exchange between different systems and components. This approach enhances system interoperability and flexibility.
